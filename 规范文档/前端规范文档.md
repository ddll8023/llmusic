# 前端规范索引

## 1. 技术栈基础

| 技术 | 版本要求 | 说明 |
|------|---------|------|
| **Vue 3** | — | Composition API + `<script setup>`，不使用 Options API |
| **Pinia** | — | 状态管理 |
| **Vue Router** | — | 路由管理 |
| **Axios** | — | HTTP 请求，需封装统一拦截器 |
| **Tailwind CSS v4** | — | 原子化 CSS，CSS-first 配置，`@tailwindcss/vite` 插件集成 |
| **TypeScript** | 可选 | 如启用，见 frontend-spec-ts skill 的 TypeScript 工具链配置 |

### 样式方案

- **Tailwind CSS v4**：使用 `@import "tailwindcss"` 引入，通过 `@theme` 指令配置主题变量
- **CSS 变量**：定义主题色、间距等全局常量
- **响应式设计**：基于 Tailwind 的响应式断点（`sm:`、`md:`、`lg:`、`xl:`）

### 组件库规范

**Element Plus 禁用规定**：项目已全面迁移到自定义组件，**全面禁止使用 Element Plus 组件**。

- **图标**：统一使用 FontAwesome 6（见代码质量与图标规范 reference）
- **UI 组件**：使用自定义 Vue 组件 + Tailwind CSS 实现
- **过渡动画**：使用 Vue 内置 `<Transition>` 组件 + Tailwind 类
- **表单验证**：使用自定义校验逻辑或 VeeValidate

---

## 2. 全局速查表

1. **禁止 Element Plus**
2. **Composition API**：使用 `<script setup>`，不使用 Options API
3. **图标统一 FontAwesome**：禁止其他图标库
4. **样式优先 Tailwind**：原子类 > `@apply` 提取 > 自定义 CSS
5. **API 参数 snake_case**：请求参数/请求体必须用 snake_case
6. **统一响应格式**：后端返回 `{code, message, data}`，code===0 为成功
7. **路由懒加载**：所有路由组件使用 `() => import()` 动态导入
8. **Props 对象语法**（JS）或 **Props 泛型语法**（TS）：见对应 skill 规范
9. **Emits 命名 kebab-case**：如 `update:modelValue`、`page-change`
10. **语义化标签**：使用 header/main/section/footer，模板层级不超过 5 层

## 4. Tailwind CSS v4 配置参考

### 入口方式

```css
/* src/style.css */
@import "tailwindcss";

@theme {
  --color-primary: #F5A623;
  --color-primary-light: #FDEBD0;
  --color-primary-dark: #D4890A;
  --color-bg: #FAFAF8;
  --color-surface: #FFFFFF;
  --color-text: #2D2D2D;
  --color-text-secondary: #999999;
  --color-text-tertiary: #BBBBBB;
  --color-border: #EBEBE7;
  --color-hover: #F5F5F2;
  --color-success: #52C41A;
  --color-error: #FF4D4F;
}
```

### Vite 配置

```typescript
// vite.config.ts
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [vue(), tailwindcss()],
})
```

### 与 v3 的核心差异

| 项目 | Tailwind v3 | Tailwind v4 |
|------|------------|-------------|
| 配置文件 | `tailwind.config.js` | 无，全部在 CSS 中 |
| PostCSS | 需要 `postcss.config.js` | 不需要 |
| Vite 集成 | `@tailwindcss/postcss` | `@tailwindcss/vite` 插件 |
| 主题定制 | `config.theme.extend` | CSS `@theme` 指令 |
| 入口方式 | `@tailwind base/components/utilities` | `@import "tailwindcss"` |

> 详细布局模式、状态交互、`@apply` 提取公共样式等，见对应 skill 的 `03-tailwind.md`。

## 5. 页面组织规范

### 目录结构

```
src/views/                    # 页面组件目录
├── home/                     # 按模块划分
│   └── index.vue             # 首页
├── user/                     # 用户模块
│   ├── Login.vue
│   ├── Register.vue
│   └── Profile.vue
└── admin/                    # 管理模块
    ├── Dashboard.vue
    └── components/           # 页面私有组件
```

### 页面文件命名

- 使用 **PascalCase**：`UserProfile.vue`、`ExamManage.vue`
- 避免使用 `index.vue` 作为页面入口（除模块根目录外）

> 详细组件目录结构与命名规则（`Base`/`App`/`Custom` 前缀），见对应 skill 的 `01-tech-stack-and-template.md`。

## 7. API 数据格式约定

为保持与后端数据库、SQLModel、FastAPI 的一致性，前端 API 调用需遵循以下约定：

| 场景 | 命名规范 | 示例 |
|------|---------|------|
| GET 请求参数 | snake_case | `{ question_type: 'exam' }` |
| POST 请求体 | snake_case | `{ subject_id: 1, order_num: 10 }` |
| PUT/PATCH 请求体 | snake_case | `{ category_name: '数据结构' }` |
| 响应数据处理 | 前端内部转换 | 接收 snake_case，组件内部使用 camelCase |

> Axios 封装、接口定义规范、页面中调用 API 的完整示例，见对应 skill 的 `05-api.md`。
