## ⚠️ 强制规则（违反即违规）

### R-1：第零层强制读取规则

> **任何任务开始前，必须使用 Read 工具完整读取以下文件（非摘要，非缓存，非系统提示）：**

| 文件 | 路径 | 读取时机 |
|------|------|---------|
| 项目 CLAUDE.md | `D:\demo\electron\llmusic\CLAUDE.md` | 每次任务 |
| 编程工作流 | `D:\demo\electron\llmusic\规范文档\编程工作流.md` | 每次任务 |
| 前端页面规范 | `D:\demo\electron\llmusic\规范文档\前端页面规范.md` | 涉及前端 UI 时 |

> **违规判定：**
> - ❌ 只看系统提示中的摘要 → 违规 (V-1)
> - ❌ 跳过第零层直接分析 → 违规 (V-1)
> - ❌ 用 Bash cat 命令代替 Read → 违规 (V-7)
> - ❌ 假设"已经知道内容"就不读 → 违规 (V-1)
> - ❌ 看到错误日志就立即开始分析代码 → 违规 (V-1)

### R-2：Skill 优先调用规则（最重要！）

> **第零层读取完成后，必须立即调用对应 Skill，Skill 指导整个分析过程：**

```
第零层读取完成
       ↓
立即调用 Skill（第零层读取后第一步！）
       ↓
按照 Skill 的 Phase 执行分析
       ↓
Skill 指导完成后 → 展示方案
       ↓
等待"嘻嘻" → 执行代码
```

> **违规判定：**
> - ❌ 先分析再调用 Skill → 违规 (V-3)
> - ❌ 不调用 Skill 直接分析 → 违规 (V-3)
> - ❌ 跳过 Skill 的 Phase 步骤 → 违规 (V-2)

### R-3：流程完整性规则

> **思考链六层必须按顺序完整执行，不可跳过、不可合并、不可简化：**

```
第零层 → 第一层 → 第二层 → 第三层 → 第四层 → 第五层 → 第六层
  ↓         ↓         ↓         ↓         ↓         ↓        ↓
 读取    理解意图   任务分类   识别阶段   问题细分   调用Skill  确认方案
  ↑
  在此处调用 Skill（第零层之后立即调用！）
```

> **违规判定：**
> - ❌ 跳过任意一层 → 违规 (V-2)
> - ❌ 合并两层为一步 → 违规 (V-2)
> - ❌ 因为"任务简单"就简化 → 违规 (V-2)

### R-4：安全词规则

> **只有收到"嘻嘻"才能执行代码修改：**

| 用户输入 | 判定结果 | 执行动作 |
|---------|---------|---------|
| `"嘻嘻"` | ✅ 安全确认 | **执行代码修改** |
| `"好的"/"可以"/"同意"` | ⚠️ 确认方案 | 继续等待"嘻嘻" |
| 其他任何内容 | ❌ 未确认 | **不执行代码** |

> **违规判定：**
> - ❌ 没有收到"嘻嘻"就执行代码 → 严重违规 (V-4)
> - ❌ 用户说"好的"就直接执行 → 违规 (V-4)

## 🔴 失败模式识别（常见违规行为）

**当你发现自己有以下想法时，立即停止并返回第零层：**

| 失败模式 | 正确行为 |
|---------|---------|
| "这个问题简单，不需要读规范" | 必须读取，即使问题看起来简单 |
| "我先看看代码，再调用 Skill" | 第零层之后立即调用 Skill |
| "我直接修复这个问题" | 必须经过完整的分析和方案确认 |
| "用户说的很清楚了，直接做" | 必须完成思考链前两层 |
| "先展示方案，再补调用 Skill" | 必须先调用 Skill，再展示方案 |

## R-7：违规检测与处理

### 违规判定

| 违规编号 | 违规行为 | 严重程度 | 处理方式 |
|---------|---------|---------|---------|
| V-1 | 跳过第零层未读取文档 | 🔴 严重 | 立即停止，返回第零层 |
| V-2 | 跳过思考链任意一层 | 🔴 严重 | 立即停止，完成缺失步骤 |
| V-3 | 未调用 Skill 就展示方案 | 🔴 严重 | 立即停止，调用 Skill 后重新分析 |
| V-4 | 未收到"嘻嘻"就执行代码 | 🔴 严重 | 立即停止，等待确认 |
| V-5 | 跳过 GAN 评估 | 🟠 中等 | 回退完成评估 |
| V-6 | 未按方案执行（擅自修改） | 🟠 中等 | 重新确认方案 |
| V-7 | 使用 Bash 代替 Read 工具 | 🟡 轻微 | 使用 Read 工具重读 |

### 违规处理流程

```
检测到违规
    ↓
标记违规编号（如：V-1 第零层读取违规）
    ↓
┌─────────────────────────────────────────┐
│ 立即停止当前操作                         │
│ 返回第零层重新执行                       │
└─────────────────────────────────────────┘
    ↓
向用户说明违规原因
    ↓
完成完整流程后再继续
```

## R-8：问答类任务简化规则

### 允许的简化
- ✅ 跳过实现步骤（因为只是查询）
- ✅ 跳过 GAN 评估（因为不涉及代码修改）
- ✅ 直接展示答案（但必须先完成思考链前两层）

### 不允许的简化
- ❌ 跳过第零层读取
- ❌ 跳过用户意图理解
- ❌ 不做任何分析就回答
- ❌ 跳过任务类型识别
- ❌ 先分析后调用 Skill

## 规范要求

1. **必须逐层分析** - 不能跳过思考链
2. **执行前必读** - 先读 doc/ 和 规范文档/
3. **Skill 优先** - 第零层之后立即调用 Skill
4. **避免降级** - 不用兜底方案、临时补丁
5. **强制安全词** - 只有"嘻嘻"触发代码执行
6. **方案先确认** - 展示方案后再执行
7. **GAN 评估** - 代码修改后必须进行三层评估
