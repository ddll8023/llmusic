# LLMusic 项目设计文档

> 本文档定义了 LLMusic 的技术架构、模块设计和数据模型。

---

## 1. 技术架构概述

### 1.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 主框架 | Electron | 跨平台桌面应用框架 |
| 前端 | Vue.js 3 + Vite | 响应式用户界面 |
| 状态管理 | Pinia | Vue 官方推荐状态管理 |
| 样式 | SCSS | 样式预处理 |
| 后端 | Python (Flask) | 凭证管理和辅助功能 |
| 数据库 | lowdb | JSON 文件数据库 |
| 音频处理 | music-metadata, fluent-ffmpeg | 元数据解析和音频处理 |

### 1.2 架构特点

```
┌─────────────────────────────────────────────────────────┐
│                    渲染进程 (Renderer)                   │
│                    Vue.js 3 + Pinia                    │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────┐  │
│  │   页面组件   │ │   通用组件   │ │   系统组件      │  │
│  └─────────────┘ └─────────────┘ └─────────────────┘  │
└───────────────────────────┬─────────────────────────────┘
                            │ IPC
┌───────────────────────────┴─────────────────────────────┐
│                      主进程 (Main)                     │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────┐  │
│  │  窗口管理   │ │  音频处理   │ │   数据管理      │  │
│  └─────────────┘ └─────────────┘ └─────────────────┘  │
└───────────────────────────┬─────────────────────────────┘
                            │ HTTP
┌───────────────────────────┴─────────────────────────────┐
│                    Python 后端                         │
│                    Flask API                          │
│  ┌─────────────┐ ┌─────────────┐                     │
│  │   凭证管理   │ │   业务逻辑   │                     │
│  └─────────────┘ └─────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

## 3. 模块设计

| `ui.js` | UI 状态（侧边栏宽度等） |

#### 3.1.2 组件分类

| 分类 | 说明 |

| `common/` | 通用可复用组件 |
| `custom/` | 自定义基础组件 |
| `pages/` | 页面级组件 |
| `system/` | 系统级组件（标题栏、播放栏） |

### 3.2 主进程模块

### 3.3 Python 后端模块

#### 3.3.1 Flask 蓝图

| 蓝图 | 职责 |

#### 3.3.2 credential 模块说明

credential 模块负责第三方服务的凭证管理：

| 模块 | 文件 | 职责 |
|------|------|------|

## 6. 关键实现说明

### 6.1 虚拟滚动

使用 `vue-virtual-scroller` 实现海量歌曲列表的流畅渲染，避免 DOM 节点过多导致的性能问题。

### 6.2 歌词同步

- 支持 `.lrc` 格式歌词文件
- 支持嵌入式歌词标签 (ID3v2 USLT)
- 提供歌词时间偏移调整功能

### 6.3 播放模式

| 模式 | 行为 |
|------|------|
| 顺序播放 | 按列表顺序播放，到末尾从头开始 |
| 随机播放 | 加权随机算法，播放次数少的歌曲权重高 |
| 单曲循环 | 重复播放当前歌曲 |

### 6.4 状态持久化

- 播放器状态（播放列表、播放模式、音量）保存到 `localStorage`
- 歌曲数据、歌单保存到 `lowdb` JSON 数据库

## 8. 错误处理设计

### 8.1 错误分类

| 类别 | 说明 | 处理方式 |
|------|------|----------|
| 文件不存在 | 歌曲文件被移动或删除 | 从数据库移除，提示用户 |
| 音轨损坏 | 音频文件损坏无法解析 | 跳过该文件，记录日志 |
| 数据库异常 | JSON 文件损坏或读写失败 | 备份恢复，重建索引 |
| 网络超时 | Python 后端请求超时 | 重试机制，超时提示 |
| 权限错误 | 无权限访问文件/文件夹 | 提示用户检查权限 |

### 8.2 文件不存在处理

```javascript
// songHandlers.js
async function getSongById(id) {
  try {
    const song = await db.getSongById(id);
    if (!song) {
      return { success: false, error: '歌曲不存在' };
    }
    // 检查文件是否存在
    const fs = require('fs');
    if (!fs.existsSync(song.filePath)) {
      console.warn(`歌曲文件不存在: ${song.filePath}`);
      // 可选：自动从数据库移除
      await db.deleteSong(id);
      return { success: false, error: '歌曲文件已被删除或移动' };
    }
    return { success: true, song };
  } catch (error) {
    return { success: false, error: error.message };
  }
}
```

### 8.3 音轨损坏处理

```javascript
// MusicScanner.js
async function parseAudioFile(filePath) {
  try {
    const metadata = await parseFile(filePath);
    return metadata;
  } catch (error) {
    console.error(`解析音频文件失败: ${filePath}`, error);
    // 返回 null，由调用方决定是跳过还是重试
    return null;
  }
}
```

### 8.4 数据库异常处理

| 场景 | 处理策略 |
|------|----------|
| JSON 解析失败 | 自动备份原文件，尝试从备份恢复 |
| 写入冲突 | 使用 lowdb 的原子写入机制 |
| 索引不一致 | 提供 `rebuildIndices()` 方法重建内存索引 |
| 数据库文件丢失 | 使用默认空数据结构初始化 |

```javascript
// Database.js
async function initDb() {
  try {
    await db.read();
    buildIndices();
  } catch (error) {
    console.error('数据库初始化失败:', error);
    // 备份损坏的文件
    const backupPath = dbPath + '.backup.' + Date.now();
    await fs.rename(dbPath, backupPath);
    // 使用默认数据重新初始化
    await db.write();
  }
}
```

### 8.5 网络超时处理

```python
# request_credential.py
import requests

def request_credential_with_timeout(url, timeout=10):
    try:
        response = requests.get(url, timeout=timeout)
        return {'success': True, 'data': response.json()}
    except requests.Timeout:
        return {'success': False, 'error': '请求超时，请稍后重试'}
    except requests.RequestException as e:
        return {'success': False, 'error': f'网络错误: {str(e)}'}
```

### 8.6 错误码定义

| 错误码 | 说明 | HTTP 状态码 |
|--------|------|-------------|
| `FILE_NOT_FOUND` | 文件不存在 | 404 |
| `FILE_CORRUPTED` | 文件损坏 | 422 |
| `DB_ERROR` | 数据库错误 | 500 |
| `NETWORK_TIMEOUT` | 网络超时 | 504 |
| `PERMISSION_DENIED` | 权限不足 | 403 |
| `INVALID_PARAM` | 参数错误 | 400 |
