# LLMusic 项目结构文档

## 1. 技术栈

| 类别 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Electron | 桌面窗口壳、系统托盘、文件系统访问、子进程管理 |
| 前端框架 | Vue 3 | UI 渲染，Composition API + `<script setup lang="ts">` |
| 前端构建 | Vite 6 | 开发服务器与构建打包 |
| 类型系统 | TypeScript 5 | **全栈渐进迁移，strict 模式**。第一阶段前端 JS→TS（vue-tsc 检查），**第二阶段 Electron 主进程 JS→TS（tsc 编译到 dist-ts/）** |
| 前端状态管理 | Pinia | 全局状态管理，类型化 Store |
| 前端 UI | Font Awesome 4 | 图标库 |
| 前端样式 | Tailwind CSS v4 | 原子化样式，`@tailwindcss/vite` 插件处理 |
| 前端 HTTP | axios | 调用后端 FastAPI 接口 |
| Electron 音频处理 | music-metadata | 读取本地音频元数据 |
| Electron 音频处理 | fluent-ffmpeg + ffmpeg-static + ffprobe-static | 音频解码与转码、封面提取、标签编辑 |
| Electron 音频处理 | node-id3 | MP3/FLAC 音频标签写入（ID3v2.4，含 USLT 歌词/APIC 封面） |
| Electron 本地数据 | lowdb | JSON 文件数据库，管理歌曲/音乐库/歌单/设置 |
| 后端框架 | FastAPI + Uvicorn | RESTful API 服务 |
| 后端序列化 | Pydantic + Pydantic Settings | 请求/响应校验与配置管理 |
| 在线音乐 SDK | qqmusic-api-python | QQ 音乐搜索、歌曲详情、歌单、下载链接、二维码登录 |
| 后端 HTTP | httpx | 解析 QQ 音乐分享链接重定向 |
| Python 依赖管理 | uv + pyproject.toml | 虚拟环境与依赖管理，位于 `backend/.venv/` |
| 打包工具 | electron-builder + PyInstaller | 桌面应用打包与 Python 后端打包为 exe |

## 3. 启动边界

- 全栈开发启动：`start-dev.bat` 或 `npm run dev`（调用 sys_electron 的 dev-runner，单终端依次启动后端 → Vite → Electron）
- 后端独立调试：`npm run dev:backend`（或 `uv run --directory backend python -m app.main`）
- 仅前端 + Electron：`npm run dev:electron`（需先 `tsc` 编译，后端需已外部启动）
- Electron 类型检查：`npm --prefix sys_electron run type-check`（`tsc --noEmit`）
- 前端类型检查：`npm --prefix sys_vue run type-check`（`vue-tsc --noEmit`）
- 后端运行在 `http://127.0.0.1:9752`，前端 Vite 开发服务器运行在 `http://127.0.0.1:9753`

## 4. 后端架构

后端采用 FastAPI 四层架构：

```
API (app/api/) → Services (app/services/) → qqmusic_api (外部 SDK)
   ↓
Schemas (app/schemas/)  ← 数据校验与序列化
Core (app/core/)        ← 配置管理
Utils (app/utils/)      ← 工具与异常
```

- **API 层**：接收请求、参数校验、调用 Service、返回响应。统一在 `qqmusic.py` 中管理，含歌曲搜索、登录认证、用户歌单、**下载元数据包**四个模块。
- **Service 层**：业务逻辑，调用 `qqmusic_api` 获取数据。`qqmusic.py` 负责歌曲搜索/详情/链接/用户歌单/**下载元数据包**，`auth.py` 负责二维码登录会话管理。
- **Schema 层**：Pydantic 模型，定义请求/响应数据结构。
- **Core 层**：应用配置（`Pydantic Settings`），支持环境变量覆盖。

## 5. 迁移说明

- 原 `python/` 已迁移到 `backend/`。
- 原 `src-renderer/` 已迁移到 `sys_vue/`。
- 原 `src-main/` 已迁移到 `sys_electron/`。
- 根目录 `package.json` 仅保留调度脚本；前端和 Electron 依赖分别声明在各自子项目中。
- **后端已从 Flask 重构为 FastAPI**（四层架构：API → Services → Schemas → Core）。
- **后端路由已统一**：`song.py`/`auth.py` 合并为 `qqmusic.py`，前缀统一为 `/api/v1/qqmusic/`。
- **前端发现音乐页面已完成重写**，接入真实后端 API，支持在线搜索、试听、下载、批量下载。
- **Electron 已新增下载功能**（`handlers/download/`），通过 IPC 通道实现文件保存，支持单曲和批量下载。
- **后端新增 QQ 音乐登录认证模块**，支持二维码登录（QQ/微信），凭证持久化与状态管理。
- **前端新增歌词全屏页面**（`LyricPage.vue`），支持 LRC 歌词解析、自动滚动、时间跳转。
- **前端样式已从 SCSS 完全迁移到 Tailwind CSS v4**，所有组件使用 Tailwind 原子类，已移除 `sass` 依赖。
- **前端已从 JavaScript 迁移到 TypeScript**：`sys_vue/src/` 下全部 `.js` → `.ts`，27 个 `.vue` 全面启用 `lang="ts"`，新增集中类型定义 `types/` 目录，构建流程集成 `vue-tsc --noEmit` 类型检查。
- **侧边栏已重构**：平台区动态渲染（当前 QQ 音乐），本地音乐与播放列表统一归类，支持设置页显隐控制。
- **在线播放队列已统一**：playerStore 新增 `onlinePlayQueue`/`onlinePlayIndex` 在线队列管理，`playNext`/`playPrevious` 统一处理本地和在线两种模式（顺序/随机/单曲循环），通过 `window._playOnlineUrl` 桥接直接控制 HTML5 Audio，消除 watcher 时序竞争。
