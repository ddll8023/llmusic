# 项目结构文档

## 1. 技术栈

- 桌面端：Electron，主进程位于 `sys_electron/`。
- 前端：Vue 3 + Vite + Pinia，渲染进程位于 `sys_vue/`。
- 前端 UI：当前仍沿用 Element Plus、Font Awesome 与项目自定义组件。
- 前端样式：当前仍沿用 SCSS 变量体系。
- 本地数据：Electron 主进程使用 lowdb 管理本地歌曲、音乐库、歌单和设置数据。
- 音频处理：`music-metadata`、`fluent-ffmpeg`、`ffmpeg-static`、`ffprobe-static`。
- 后端：Flask + Flask-CORS，后端服务位于 `backend/`。
- 在线音乐数据：`qqmusic-api-python`、`httpx`、`asyncio`。
- Python 依赖管理：新增 `backend/pyproject.toml`，用于后续通过 uv 管理后端依赖。

## 3. 启动边界

- 后端单独启动：`start_backend.bat`
- 桌面端启动：`start_electron.bat`
- `sys_electron` 的 `dev` 脚本会同时启动 `sys_vue` 的 Vite 开发服务器和 Electron。
- 当前迁移未执行依赖安装、构建、测试或服务启动。

## 4. 迁移说明

- 原 `python/` 已迁移到 `backend/`。
- 原 `src-renderer/` 已迁移到 `sys_vue/`。
- 原 `src-main/` 已迁移到 `sys_electron/`。
- 根目录 `package.json` 仅保留调度脚本；前端和 Electron 依赖分别声明在各自子项目中。
- 当前前端仍保留原组件组织方式，尚未重构为 `views/router/api/stores` 的完整独立 Vue 应用结构。
- 当前后端仍保留 Flask 实现，尚未重构为 FastAPI。
