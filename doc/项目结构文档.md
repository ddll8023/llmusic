# 项目结构文档

## 1. 技术栈

- 桌面端：Electron，主进程位于 `sys_electron/`。
- 前端：Vue 3 + Vite + Pinia，渲染进程位于 `sys_vue/`。
- 前端 UI：当前仍沿用 Element Plus、Font Awesome 与项目自定义组件。
- 前端样式：Tailwind CSS v4 原子化样式（已从 SCSS 完全迁移），主题定义在 `@theme` 中。
- 本地数据：Electron 主进程使用 lowdb 管理本地歌曲、音乐库、歌单和设置数据。
- 音频处理：`music-metadata`、`fluent-ffmpeg`、`ffmpeg-static`、`ffprobe-static`。
- 后端：FastAPI + Uvicorn + Pydantic，后端服务位于 `backend/`。
- 在线音乐数据：`qqmusic-api-python`、`httpx`、`asyncio`。
- HTTP 客户端（前端）：`axios`。
- Python 依赖管理：`pyproject.toml` + `uv`，虚拟环境位于 `backend/.venv/`。

## 3. 启动边界

- 后端启动（Linux）：`start_backend.sh`，或 `uv run --directory backend python -m app.main`
- 后端启动（Windows）：`start_backend.bat`
- 桌面端启动（Linux）：`start_electron.sh`
- 桌面端启动（Windows）：`start_electron.bat` 或 `start.bat`
- `sys_electron` 的 `dev` 脚本会同时启动 `sys_vue` 的 Vite 开发服务器和 Electron。
- 后端运行在 `localhost:9752`，前端 Vite 开发服务器运行在 `localhost:9753`。

## 4. 后端架构

后端采用 FastAPI 四层架构：

```
API (app/api/) → Services (app/services/) → qqmusic_api (外部 SDK)
   ↓
Schemas (app/schemas/)  ← 数据校验与序列化
Core (app/core/)        ← 配置管理
Utils (app/utils/)      ← 工具与异常
```

- **API 层**：接收请求、参数校验、调用 Service、返回响应。
- **Service 层**：业务逻辑，调用 `qqmusic_api` 获取数据。
- **Schema 层**：Pydantic 模型，定义请求/响应数据结构。
- **Core 层**：应用配置（`Pydantic Settings`）。

## 5. 迁移说明

- 原 `python/` 已迁移到 `backend/`。
- 原 `src-renderer/` 已迁移到 `sys_vue/`。
- 原 `src-main/` 已迁移到 `sys_electron/`。
- 根目录 `package.json` 仅保留调度脚本；前端和 Electron 依赖分别声明在各自子项目中。
- **后端已从 Flask 重构为 FastAPI**（四层架构：API → Services → Schemas → Core）。
- **前端发现音乐页面已完成重写**，接入真实后端 API，支持在线搜索、试听、下载。
- **Electron 已新增下载功能**（`handlers/download/`），通过 IPC 通道实现文件保存。
- **前端样式已从 SCSS 完全迁移到 Tailwind CSS v4**，所有组件使用 Tailwind 原子类，已移除 `sass` 依赖。
- 前端尚未完全重构为 `views/router/api/stores` 的独立 Vue 应用结构。
