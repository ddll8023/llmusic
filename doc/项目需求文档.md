# LLMusic 项目需求文档

> 本文档定义了 LLMusic 本地音乐播放器的功能需求和非功能需求。

---

## 1. 文档概述

### 1.1 项目背景

LLMusic 是一款基于 Electron、Vue.js 和 Python 构建的桌面音乐播放器，专注于本地音乐文件的管理与播放。项目采用混合架构，前端使用 Vue.js 3 构建用户界面，主进程处理系统级操作，Python 后端提供凭证管理等辅助功能。

### 1.2 项目目标

- 提供流畅的本地音乐播放体验
- 支持多音乐库管理
- 实现歌曲元数据和歌词的自动解析
- 提供良好的用户交互和视觉体验

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 音乐库管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 添加音乐库 | 用户可添加本地文件夹作为音乐库 | P0 |
| 切换音乐库 | 支持多个音乐库快速切换 | P0 |
| 扫描音乐 | 自动扫描音乐库中的音频文件 | P0 |
| 移除音乐库 | 删除音乐库及其关联歌曲 | P1 |

#### 2.1.2 音乐播放

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 播放控制 | 播放、暂停、上一首、下一首 | P0 |
| 进度控制 | 进度条拖拽、音量调节 | P0 |
| 播放模式 | 顺序播放、随机播放、单曲循环 | P0 |
| 播放列表 | 支持播放队列管理 | P0 |

#### 2.1.3 元数据管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 自动解析 | 解析歌曲标题、艺术家、专辑、时长 | P0 |
| 封面显示 | 提取并显示专辑封面 | P0 |
| 歌词同步 | 支持 .lrc 格式歌词 | P1 |
| 标签编辑 | 支持编辑歌曲元数据 | P2 |

#### 2.1.4 歌单管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建歌单 | 创建自定义播放列表 | P1 |
| 编辑歌单 | 重命名、添加/移除歌曲 | P1 |
| 删除歌单 | 删除播放列表 | P1 |

#### 2.1.5 系统托盘

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 最小化到托盘 | 关闭窗口时最小化到系统托盘 | P1 |
| 托盘菜单 | 右键显示播放控制菜单（播放/暂停、上一首、下一首、退出） | P1 |
| 托盘图标 | 显示应用图标，双击还原窗口 | P1 |
| 状态同步 | 托盘图标反映当前播放状态 | P2 |

### 2.2 用户界面功能

#### 2.2.1 页面模块

| 页面 | 描述 |
|------|------|
| 音乐库 | 展示音乐库中的歌曲列表 |
| 发现音乐 | 推荐/发现页（预留） |
| 歌词页面 | 全屏歌词展示 |
| 设置页面 | 应用设置 |
| 元数据管理 | 批量管理歌曲信息 |

#### 2.2.2 系统组件

| 组件 | 描述 |
|------|------|
| 标题栏 | 自定义窗口标题栏 |
| 侧边栏 | 导航菜单 |
| 播放栏 | 底部播放控制条 |
| 歌曲表格 | 虚拟滚动歌曲列表 |
| 上下文菜单 | 右键操作菜单 |

## 4. 用户场景

### 4.1 场景一：首次使用

1. 用户启动应用
2. 弹出添加音乐库提示
3. 用户选择本地音乐文件夹
4. 系统扫描并解析歌曲
5. 显示歌曲列表

### 4.2 场景二：日常播放

1. 用户从歌曲列表选择歌曲
2. 开始播放，显示播放栏
3. 用户可切换上一首/下一首
4. 可拖拽进度条调整播放位置

### 4.3 场景三：歌词查看

1. 用户点击歌词按钮
2. 切换到歌词全屏页面
3. 歌词随播放进度滚动
4. 可手动调整歌词同步

## 5. 术语表

| 术语 | 说明 |
|------|------|
| 音乐库 (Library) | 用户添加的本地音乐文件夹 |
| 歌曲 (Song) | 单个音频文件及其元数据 |
| 歌单 (Playlist) | 用户创建的音乐播放列表 |
| IPC | 进程间通信 (Inter-Process Communication) |
| 元数据 (Metadata) | 歌曲的标题、艺术家、专辑等信息 |
| LRC | 歌词文件格式 |
