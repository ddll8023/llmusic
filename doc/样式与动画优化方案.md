# LLMusic 样式与动画优化方案

> 参考 [Mineradio](https://github.com/XxHuberrr/Mineradio) 沉浸式音乐播放器的视觉设计语言，对 LLMusic 的样式和动画进行系统性优化重构。
>
> 本文档记录优化项清单、优先级、涉及文件、实施进展及验收标准。更新日期：2026-07-01

---

## 1. 现状与目标

### 1.1 当前问题总结

通过全面审查 LLMusic 的样式实现，发现以下系统性问题：

| 问题类别 | 严重程度 | 状态 | 说明 |
|---------|:--------:|:----:|------|
| 玻璃阴影重复定义 | 高 | ✅ P0-1 已修复 | 同一组值在 4 处重复，已收归 CSS 变量单源管理 |
| 缓动函数不统一 | 高 | ⏳ 待实施 | 项目散落 4 种不同缓动函数 |
| 过渡时长不统一 | 高 | ⏳ 待实施 | 散落 150/200/250/300/350ms 五种时长 |
| 动画时长 CSS 变量未落地 | 高 | ⏳ 待实施 | `--duration-*` 定义了但无组件引用 |
| 出场动画缺失 | 中 | ⏳ 待实施 | ContextMenu/CustomModal 关闭时瞬间消失 |
| 硬编码颜色未使用 tokens | 中 | ⏳ 待实施 | PlayerBar/LyricPage 多处使用 `#535353`、`#a0a0a0` 等 |
| `transition-all` 滥用 | 中 | ⏳ 待实施 | 多处使用 `transition-all` 应限定属性 |
| 玻璃质感无 RGB 色差 | 中 | ⏳ 待实施 | 仅普通 `backdrop-filter`，无色差扭曲 |
| 侧边栏无玻璃质感 | 中 | ⏳ 待实施 | 纯色 surface 与玻璃风格的 PlayerBar 视觉断裂 |
| 选中态无过渡动画 | 低 | ⏳ 待实施 | SideBar/Btn 选中切换瞬间跳变 |
| 按钮无 active 态 | 低 | ⏳ 待实施 | 缺少按下反馈 |
| 无入场动画 | 低 | ⏳ 待实施 | 搜索/翻页/结果切换无过渡 |
| DOM 直接操作非 Vue 风格 | 低 | ⏳ 待实施 | MusicLibrary 使用 `classList`/`textContent` |

### 1.2 优化目标

1. **单源玻璃定义** ✅（P0-1 已完成）：消除玻璃阴影的重复定义，集中管理
2. **统一动效语言**：全局统一缓动函数和过渡时长体系
3. **设计 token 落地**：所有硬编码值替换为 `var(--color-*)` / `var(--glass-*)`
4. **动画完整闭环**：补全入场动画、出场动画、交互反馈
5. **视觉深度提升**：引入 Mineradio 风格的 RGB 色差玻璃和氛围色系统

## 2. 优化项清单

### P0 — 高优先级，低成本

#### P0-1：玻璃阴影单源化 ✅ 已完成

- **问题**：玻璃阴影值在 `tailwind-entry.css`、`useGlassStyles.ts`、`GlassButton.vue`、`PlayerBar.vue` 四处重复硬编码
- **方案（对抗审查后修正）**：
  - 玻璃变量移出 `@theme` 到独立 `:root` 块，职责分离
  - **保留 8 个原子变量**（`--glass-highlight`, `--glass-glow`, `--glass-outer-*`, `--glass-inset-*`），供组合变量引用
  - 新增完整组合变量 `--glass-panel-shadow`，通过 `var()` 引用原子变量构建，建立引用链
  - PlayerBar 背景色保留 `.12`，通过独立变量 `--glass-playerbar-bg` 隔离
  - 删除 `useGlassStyles.ts` 中的虚代码 `shadowLayers` 和 `paddingMap`
- **实际变更**：

  | 文件 | 修改内容 |
  |------|---------|
  | `tailwind-entry.css` | 8 个原子变量保留 + 新增组合变量（`--glass-panel-shadow` 等）+ 新增 `--glass-playerbar-bg`；移出 `@theme` 到独立 `:root` |
  | `useGlassStyles.ts` | 所有硬编码阴影/背景 → `var(--glass-*)`；删除 `shadowLayers`；删除 `paddingMap`；代码量减少约 40% |
  | `GlassButton.vue` | 6 处 Tailwind 任意值 → `var(--glass-bg/btn-shadow/btn-hover-shadow/btn-hover-bg/blur/saturate/brightness/radius)` |
  | `PlayerBar.vue` | `background`、`backdrop-filter`、`box-shadow`、hover `box-shadow` 全部 → `var(--glass-*)` |
- **涉及文件**（共 4 个）：
  - `sys_vue/src/styles/tailwind-entry.css`
  - `sys_vue/src/composables/useGlassStyles.ts`
  - `sys_vue/src/components/common/GlassButton.vue`
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验证方式**：修改 `tailwind-entry.css` 中任一原子变量，所有组件玻璃阴影同步变化

#### P0-2：缓动函数统一

- **问题**：混合使用 `cubic-bezier(.16,1,.3,1)`、`cubic-bezier(0.4, 0, 0.2, 1)`、`ease`、`ease-out` 四种缓动
- **方案**：全局统一为 Mineradio 缓出曲线 `cubic-bezier(.16,1,.3,1)`（比 `ease-out-expo` 更具弹性感），删除 `--ease-out-expo` 变量
- **涉及文件**：
  - `sys_vue/src/styles/tailwind-entry.css`（更新 `--ease-*` 变量）
  - `sys_vue/src/styles/base.css`（更新 `.fade-*`、`.slide-fade-*` 等过渡）
  - `sys_vue/src/components/system/PlayerBar.vue`
  - `sys_vue/src/components/system/SideBar.vue`
  - `sys_vue/src/components/pages/LyricPage.vue`
  - `sys_vue/src/components/common/GlassButton.vue`
  - `sys_vue/src/components/common/ContextMenu.vue`
  - `sys_vue/src/components/custom/CustomModal.vue`
- **验收标准**：全局所有过渡均使用同一缓动曲线

#### P0-3：过渡时长变量落地

- **问题**：`--duration-fast/base/slow` 定义在 `tailwind-entry.css` 但无组件引用，时长散落五种
- **方案**：组件统一引用 `var(--duration-fast/base/slow)`
  - `--duration-fast: 150ms` → 悬停、小反馈
  - `--duration-base: 250ms` → 面板切换、默认过渡
  - `--duration-slow: 400ms` → 展开收缩、复杂过渡
- **涉及文件**：
  - 所有含 `transition`、`animation-duration` 的 `.vue` 文件
  - `sys_vue/src/styles/base.css`
- **验收标准**：全局无硬编码过渡时长值（除特殊情况外）

### P1 — 中优先级，中等成本

#### P1-1：引入 SVG RGB 色差玻璃

- **问题**：当前玻璃使用普通 `backdrop-filter: blur() saturate() brightness()`，无色差效果
- **方案**：在 `index.html` 或 `App.vue` 中定义 SVG `<filter>`，通过 `backdrop-filter: url(#id)` 在关键组件上启用 RGB 三通道位移色差扭曲
- **参考**：Mineradio 的 SVG filter 定义：
  ```svg
  <filter id="glass-filter" x="-12%" y="-28%" width="124%" height="156%">
    <!-- 红通道位移 -->  <feColorMatrix> + <feDisplacementMap>
    <!-- 绿通道位移 -->  <feColorMatrix> + <feDisplacementMap>
    <!-- 蓝通道位移 -->  <feColorMatrix> + <feDisplacementMap>
    <!-- 合并 + 偏移纠正 -->
  </filter>
  ```
- **涉及文件**：
  - `sys_vue/index.html`（注入 SVG filter 定义）
  - `sys_vue/src/styles/tailwind-entry.css`（新增 `--glass-svg-filter` 变量）
  - `sys_vue/src/composables/useGlassStyles.ts`（新增 SVG 模式支持）
  - `sys_vue/src/components/system/PlayerBar.vue`
  - `sys_vue/src/components/common/GlassPanel.vue`
  - `sys_vue/src/components/common/GlassButton.vue`
- **验收标准**：PlayerBar 启用 SVG 玻璃后产生肉眼可见的 RGB 色差扭曲，无性能明显下降
- **注意事项**：禁止替换为普通毛玻璃；禁止大面积白色渐变扫过；禁止中心糊成一团

#### P1-2：补充出场动画

- **问题**：ContextMenu、CustomModal 关闭时瞬间消失
- **方案**：使用 Vue `<Transition>` 的 `leave-active`/`leave-to` 类，为所有弹出式组件添加出场过渡
  - ContextMenu：`opacity 0 → 1` + `transform: scale(0.95)` 缩小退场
  - CustomModal：遮罩 `opacity` 过渡 + 内容 `scale + blur` 退场
- **涉及文件**：
  - `sys_vue/src/components/common/ContextMenu.vue`
  - `sys_vue/src/components/custom/CustomModal.vue`
  - `sys_vue/src/components/common/DeleteConfirmDialog.vue`
- **验收标准**：所有模态/菜单关闭时均有 200ms 平滑过渡

#### P1-3：PlayerBar box-shadow 过渡

- **问题**：`.ribbon` hover 时切换 box-shadow 但无 `transition`，瞬间跳变
- **方案**：为 `.ribbon` 添加 `transition: box-shadow var(--duration-base) var(--ease-default)`
- **涉及文件**：
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验收标准**：hover PlayerBar 时玻璃阴影平滑变化

#### P1-4：useGlassStyles.ts 重构 ⚠️ 部分已完成

- **已完成部分**（在 P0-1 中一并处理）：
  - ✅ 所有硬编码字符串 → `var(--glass-*)` 引用
  - ✅ 删除 `shadowLayers` 虚代码（`eco` 模式不再有阴影叠加问题）
  - ✅ 删除 `paddingMap`（未使用代码）
- **待完成部分**：
  - ⏳ `panel` 与 `elevated` 变体阴影值仍完全相同，需明确差异化或合并
  - ⏳ `elevated` 的 `hoverShadow`/`hoverBg` 尚未被任何组件消费
- **涉及文件**：
  - `sys_vue/src/composables/useGlassStyles.ts`

### P2 — 中优先级，低成本

#### P2-1：消除硬编码颜色

- **问题**：PlayerBar 使用 `#535353`、`#383838`，LyricPage 使用 `#a0a0a0`、`#888` 等
- **方案**：全局搜索所有 `.vue` 和 `.css` 文件中的色值，替换为设计 tokens
  - `#535353` → `var(--color-content-disabled)`
  - `#383838` → `var(--color-line-light)`
  - `#a0a0a0` → `var(--color-content-tertiary)`
  - `#888` → `var(--color-content-tertiary)`
- **涉及文件**：所有 `.vue` 和 `.css` 文件
- **验收标准**：项目中无无关的硬编码颜色值（tailwind-entry.css 中的 token 定义除外）

#### P2-2：侧边栏玻璃质感

- **问题**：SideBar 纯色 surface，与 PlayerBar 的玻璃风格视觉断裂
- **方案**：SideBar 添加弱玻璃背景（`background: rgba(26,26,26,.92)` + `backdrop-filter: blur(4px)`），底部与 PlayerBar 玻璃区域自然衔接
- **涉及文件**：
  - `sys_vue/src/components/system/SideBar.vue`
- **验收标准**：侧边栏呈现轻微玻璃质感，不与 PlayerBar 风格冲突

#### P2-3：`transition-all` 限定属性

- **问题**：GlassButton、ContextMenu、DiscoverMusic 多处使用 `transition-all`
- **方案**：限定为具体过渡属性，如 `transition-[background-color,box-shadow,transform] duration-250`
- **涉及文件**：
  - `sys_vue/src/components/common/GlassButton.vue`
  - `sys_vue/src/components/common/ContextMenu.vue`
  - `sys_vue/src/components/pages/DiscoverMusic.vue`
- **验收标准**：无 `transition-all` 滥用，性能无退化

#### P2-4：按钮 active 态

- **问题**：GlassButton、CustomButton 只有 hover 态，无按下反馈
- **方案**：添加 `active:scale-95` 微缩小 + `box-shadow` 内收，形成仿物理按压感
- **涉及文件**：
  - `sys_vue/src/components/common/GlassButton.vue`
  - `sys_vue/src/components/custom/CustomButton.vue`
- **验收标准**：按下按钮时产生 0.95 缩小的按压反馈

#### P2-5：侧边栏选中态过渡

- **问题**：SideBar 菜单 active 切换瞬间背景色跳变
- **方案**：选中态添加 `transition-[background-color,color] duration-200` 缓动过渡
- **涉及文件**：
  - `sys_vue/src/components/system/SideBar.vue`
- **验收标准**：侧边栏菜单切换时有平滑的背景色过渡

### P3 — 低优先级，高成本

#### P3-1：背景氛围粒子系统

- **问题**：纯色黑色背景缺少动态氛围
- **方案**：在 `App.vue` 或独立组件中实现 Canvas 轻量粒子背景（星星闪烁/流动），未播放时保持干净星河氛围，播放后叠加光晕
- **参考**：Mineradio 的星河背景（未播放态）+ Emily 视觉（播放态）
- **涉及文件**：新建组件或 `App.vue`
- **验收标准**：背景呈现动态粒子效果，CPU 占用 < 5%

#### P3-2：搜索翻页入场动画

- **问题**：DiscoverMusic 搜索结果切换/翻页时无过渡
- **方案**：为结果容器添加 `transition-group`，行添加 `fade-in-up` 逐个入场
- **涉及文件**：
  - `sys_vue/src/components/pages/DiscoverMusic.vue`
  - `sys_vue/src/components/business/BaseSongTable.vue`
- **验收标准**：翻页/搜索时歌曲行逐个淡入上浮

#### P3-3：LyricPage 歌词分层动画

- **问题**：当前歌词行无特殊动画，非当前行无视觉区分
- **方案**：当前行添加渐入缩放 + 辉光效果，非当前行降低 opacity + 微缩，参考 Mineradio 歌词层级（readability → glow → sun）
- **涉及文件**：
  - `sys_vue/src/components/pages/LyricPage.vue`
- **验收标准**：当前歌词行有突出动画，前后行自然淡出

#### P3-4：专辑封面色扩散系统

- **问题**：`useAlbumColors.ts` 已提取主色，但仅用于底部光晕
- **方案**：将主色扩散到侧边栏选中指示器、PlayerBar 进度条、高亮色等
- **涉及文件**：
  - `sys_vue/src/composables/useAlbumColors.ts`
  - `sys_vue/src/components/system/SideBar.vue`
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验收标准**：播放歌曲后，UI 高亮色跟随封面主色变化

#### P3-5：MusicLibrary DOM 操作修复

- **问题**：`classList.add('highlight-update')` 和 `textContent` 直接操作 DOM
- **方案**：改为响应式 `ref` + computed 驱动的 CSS class 绑定
- **涉及文件**：
  - `sys_vue/src/components/pages/MusicLibrary.vue`
- **验收标准**：无原生 DOM 操作，全响应式驱动
