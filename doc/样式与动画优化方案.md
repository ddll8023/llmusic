# LLMusic 样式与动画优化方案

> 参考 [Mineradio](https://github.com/XxHuberrr/Mineradio) 沉浸式音乐播放器的视觉设计语言，对 LLMusic 的样式和动画进行系统性优化重构。
>
> 本文档记录优化项清单、优先级、涉及文件、实施进展及验收标准。更新日期：2026-07-02（P2 五项全部完成，详见 §2）

---

## 1. 现状与目标

### 1.1 当前问题总结

通过全面审查 LLMusic 的样式实现，发现以下系统性问题：

| 问题类别 | 严重程度 | 状态 | 说明 |
|---------|:--------:|:----:|------|
| 玻璃阴影重复定义 | 高 | ✅ P0-1 已修复 | 同一组值在 4 处重复，已收归 CSS 变量单源管理 |
| 缓动函数不统一 | 高 | ⏳ 待实施 | 项目散落 4 种不同缓动函数 |
| 过渡时长不统一 | 高 | ⏳ 待实施 | 散落 150/200/250/300/350ms 五种时长 |
| 动画时长 CSS 变量未落地 | 高 | ⏳ 待实施 | `--duration-*` 定义了但无组件引用 |
| 出场动画缺失 | 中 | ✅ P1-2 已修复 | ContextMenu/CustomModal/TagEditor 关闭时平滑淡出 |
| 玻璃质感无 RGB 色差 | 中 | ✅ P1-1 已修复 | PlayerBar 叠加 SVG filter，边缘产生微弱 RGB 色差 |
| 硬编码颜色未使用 tokens | 中 | ✅ P2-1 已修复 | PlayerBar/LyricPage/SongTable/base.css 共 28 处替换为 `var(--color-*)` |
| `transition-all` 滥用 | 中 | ✅ P2-3 已修复 | GlassButton/ContextMenu(×2)/DiscoverMusic 4 处限定到具体属性 |
| 侧边栏无玻璃质感 | 中 | ✅ P2-2 已修复 | `bg-surface-elevated`→`bg-surface-sidebar`，玻璃色变量化 |
| 选中态无过渡动画 | 低 | ✅ 不存在 | 审查确认 SideBar 已有 `transition-[background-color,color] duration-150`，取消修改 |
| 按钮无 active 态 | 低 | ✅ P2-4 已修复 | GlassButton/CustomButton 添加 `active:scale-95` 按压反馈 |
| 背景无动态氛围 | 低 | ✅ P3-1 已修复 | Canvas 流动光晕背景，交互式鼠标跟随 |
| DOM 直接操作非 Vue 风格 | 低 | ⏳ 待实施 | MusicLibrary 使用 `classList`/`textContent` |

### 1.2 优化目标

1. **单源玻璃定义** ✅（P0-1 已完成）：消除玻璃阴影的重复定义，集中管理
2. **统一动效语言**：全局统一缓动函数和过渡时长体系
3. **设计 token 落地** ✅（P2-1 已完成）：所有硬编码色值替换为 `var(--color-*)` / `var(--glass-*)`
4. **动画完整闭环** ✅（P2-4 已完成）：补全入场动画、出场动画、交互反馈（按压缩放）
5. **视觉深度提升**：引入 Mineradio 风格的 RGB 色差玻璃和氛围色系统

## 2. 优化项清单

### P0 — 高优先级，低成本

#### P0-1：玻璃阴影单源化 ✅ 已完成

- **问题**：玻璃阴影值在 `tailwind-entry.css`、`useGlassStyles.ts`、`GlassButton.vue`、`PlayerBar.vue` 四处重复硬编码
- **方案（对抗审查后修正）**：
  - 玻璃变量移出 `@theme` 到独立 `:root` 块，职责分离
  - **保留 8 个原子变量**（`--glass-highlight`, `--glass-glow`, `--glass-outer-*`, `--glass-inset-*`），供组合变量引用
  - 新增完整组合变量 `--glass-panel-shadow`，通过 `var()` 引用原子变量构建，建立引用链
  - PlayerBar 背景色保留 `.12`，通过独立变量 `--glass-playerbar-bg` 隔离
  - 删除 `useGlassStyles.ts` 中的虚代码 `shadowLayers` 和 `paddingMap`
- **实际变更**：

  | 文件 | 修改内容 |
  |------|---------|
  | `tailwind-entry.css` | 8 个原子变量保留 + 新增组合变量（`--glass-panel-shadow` 等）+ 新增 `--glass-playerbar-bg`；移出 `@theme` 到独立 `:root` |
  | `useGlassStyles.ts` | 所有硬编码阴影/背景 → `var(--glass-*)`；删除 `shadowLayers`；删除 `paddingMap`；代码量减少约 40% |
  | `GlassButton.vue` | 6 处 Tailwind 任意值 → `var(--glass-bg/btn-shadow/btn-hover-shadow/btn-hover-bg/blur/saturate/brightness/radius)` |
  | `PlayerBar.vue` | `background`、`backdrop-filter`、`box-shadow`、hover `box-shadow` 全部 → `var(--glass-*)` |
- **涉及文件**（共 4 个）：
  - `sys_vue/src/styles/tailwind-entry.css`
  - `sys_vue/src/composables/useGlassStyles.ts`
  - `sys_vue/src/components/common/GlassButton.vue`
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验证方式**：修改 `tailwind-entry.css` 中任一原子变量，所有组件玻璃阴影同步变化

#### P0-2：缓动函数统一

- **问题**：混合使用 `cubic-bezier(.16,1,.3,1)`、`cubic-bezier(0.4, 0, 0.2, 1)`、`ease`、`ease-out` 四种缓动
- **方案**：全局统一为 Mineradio 缓出曲线 `cubic-bezier(.16,1,.3,1)`（比 `ease-out-expo` 更具弹性感），删除 `--ease-out-expo` 变量
- **涉及文件**：
  - `sys_vue/src/styles/tailwind-entry.css`（更新 `--ease-*` 变量）
  - `sys_vue/src/styles/base.css`（更新 `.fade-*`、`.slide-fade-*` 等过渡）
  - `sys_vue/src/components/system/PlayerBar.vue`
  - `sys_vue/src/components/system/SideBar.vue`
  - `sys_vue/src/components/pages/LyricPage.vue`
  - `sys_vue/src/components/common/GlassButton.vue`
  - `sys_vue/src/components/common/ContextMenu.vue`
  - `sys_vue/src/components/custom/CustomModal.vue`
- **验收标准**：全局所有过渡均使用同一缓动曲线

#### P0-3：过渡时长变量落地

- **问题**：`--duration-fast/base/slow` 定义在 `tailwind-entry.css` 但无组件引用，时长散落五种
- **方案**：组件统一引用 `var(--duration-fast/base/slow)`
  - `--duration-fast: 150ms` → 悬停、小反馈
  - `--duration-base: 250ms` → 面板切换、默认过渡
  - `--duration-slow: 400ms` → 展开收缩、复杂过渡
- **涉及文件**：
  - 所有含 `transition`、`animation-duration` 的 `.vue` 文件
  - `sys_vue/src/styles/base.css`
- **验收标准**：全局无硬编码过渡时长值（除特殊情况外）

### P1 — 中优先级，中等成本

#### P1-1：引入 SVG RGB 色差玻璃 ✅ 已完成

- **问题**：当前玻璃使用普通 `backdrop-filter: blur() saturate() brightness()`，无色差效果
- **方案**（对抗审查后修正）：
  - 采用**叠加策略**：保留 PlayerBar 原有的 `backdrop-filter` 毛玻璃，额外叠加 `filter: url(#mr-glass)` 实现色差
  - PlayerBar 内部添加独立 `.ribbon-glass-overlay` div 承载 SVG filter，避免影响 `::before`/`::after` 伪元素
  - SVG filter 参数适配 64px 高度：`scale` 从 180/170/160 降至 **5/4/3**，`dx` 从 -90 归零
  - filter bounds 收窄：`x="-12%" y="-28%" → x="-5%" y="-10%"`
  - 不在 `useGlassStyles.ts` 中新增选项（仅 PlayerBar 一个消费者，直接 CSS 类控制）
- **实际变更**：

  | 文件 | 修改内容 |
  |------|---------|
  | `sys_vue/index.html` | scale=180/170/160 → 5/4/3；dx=-90 → 0；bounds 收窄 |
  | `sys_vue/src/styles/tailwind-entry.css` | 新增 `--glass-svg-filter: url(#mr-glass)` 变量 |
  | `sys_vue/src/components/system/PlayerBar.vue` | 新增 `.ribbon-glass-overlay` 独立覆盖层（`filter: var(--glass-svg-filter)`, `pointer-events: none`）；保留原有 backdrop-filter |
- **涉及文件**（共 3 个）：
  - `sys_vue/index.html`
  - `sys_vue/src/styles/tailwind-entry.css`
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验收标准**：PlayerBar 保留原有毛玻璃基础上叠加微弱 RGB 边缘色差（微妙的彩色分离）

#### P1-2：补充出场动画（含入场统一）✅ 已完成

- **问题**：ContextMenu/CustomModal/DeleteConfirmDialog 关闭时瞬间消失
- **方案**（对抗审查后修正）：
  - 使用 Vue `<Transition>` 同时管理入场和出场动画，替换旧的 CSS `@keyframes` 类
  - 入场也从 CSS keyframe（`fade-in-up`/`modal-fade-in`）迁移到 Transition，统一动画体系
  - ContextMenu：`opacity` + `transform: scale(0.95)↔1`，时长 `var(--duration-fast)`（150ms）
  - CustomModal/DeleteConfirmDialog/TagEditor：遮罩 `opacity` 过渡 + 内容 `scale + opacity` 过渡，时长 `var(--duration-base)`（250ms）
  - 所有过渡使用 Mineradio 缓动 `cubic-bezier(.16,1,.3,1)`
- **实际变更**：

  | 文件 | 修改内容 |
  |------|---------|
  | `sys_vue/src/components/common/ContextMenu.vue` | `<Transition name="context-menu">` 包裹；删除 `fade-in-up`；scoped CSS 定义 enter/leave |
  | `sys_vue/src/components/custom/CustomModal.vue` | 双层 Transition（遮罩 `modal-overlay` + 内容 `modal-content`）；删除 `fade-in`/`modal-fade-in` |
  | `sys_vue/src/components/common/DeleteConfirmDialog.vue` | 同 CustomModal 模式；删除 `animate-modal-fade-in` |
  | `sys_vue/src/components/common/TagEditor.vue` | **新增纳入范围**，同 DeleteConfirmDialog 模式 |
  | `sys_vue/src/styles/base.css` | 清理已迁移的 `@keyframes fadeInUp`、`@keyframes modalFadeIn` 及相关类 |
- **涉及文件**（共 5 个）：
  - `sys_vue/src/components/common/ContextMenu.vue`
  - `sys_vue/src/components/custom/CustomModal.vue`
  - `sys_vue/src/components/common/DeleteConfirmDialog.vue`
  - `sys_vue/src/components/common/TagEditor.vue`
  - `sys_vue/src/styles/base.css`
- **验收标准**：所有模态/菜单关闭时有 250ms 平滑过渡（ContextMenu 为 150ms），入场出场对称

#### P1-3：PlayerBar box-shadow 过渡 ✅ 已完成

- **问题**：`.ribbon` hover 时切换 box-shadow 但无 `transition`，瞬间跳变
- **方案**（对抗审查后修正）：为 `.ribbon` 添加 `transition: box-shadow var(--duration-fast) cubic-bezier(.16,1,.3,1)`（150ms 短过渡避免 paint 开销），同步添加 `will-change: height, box-shadow` 提升合成层
- **涉及文件**：
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验收标准**：hover PlayerBar 时玻璃阴影 150ms 平滑变化，无跳变

#### P1-4：useGlassStyles.ts 重构 ✅ 已完成

- **问题**：
  - `panel` 与 `elevated` 变体阴影值完全相同
  - `elevated` 的 `hoverShadow`/`hoverBg` 未被任何组件消费
  - `hoverBg` 指向不存在的 CSS 变量（`--glass-button-hover-bg` vs `--glass-btn-hover-bg`）
  - `classes` 返回值从未被使用
- **方案**（对抗审查后修正）：
  - 删除 `elevated` 变体（从 `GlassVariant` 类型和 `variantConfigs` 中移除）
  - 删除所有变体的 `hoverShadow` 和 `hoverBg` 字段（无人消费）
  - 删除 `GlassStyles` 接口中的 `classes` 字段及其 computed/return
- **涉及文件**：
  - `sys_vue/src/composables/useGlassStyles.ts`
- **验收标准**：`GlassVariant` 精简为 `'panel' | 'button'`；`variantConfigs` 仅含 `shadow` 字段；接口干净无死代码

### P3 — 低优先级，高成本

#### P3-1：流动光晕背景 ✅ 已完成

- **问题**：纯色黑色背景缺少动态氛围
- **最终方案**：Canvas 2D 实时渲染的交互式流动光晕背景，替代原方案的粒子系统（审查后确认星尘风格不符合预期，改为光晕方案）
  - 7 个径向渐变光晕球，跟随鼠标流动，不同延迟产生流体拖尾感
  - 鼠标不动时，光晕在鼠标位置周围缓慢正弦漂移
  - `screen` 混合叠加色彩 + `hard-light` 混合增加层次
  - 底部品牌绿色光晕跟随鼠标水平移动
  - 播放态色彩强度提升 50%
  - 暗角 vignette 聚焦视觉中心
  - `prefers-reduced-motion` 支持
  - **层叠策略（Bug 修复记录）**：组件在 `#app-container` 内部 `position: absolute; z-index: -1`，移除 `#app-container` 的 `z-index` 防止层叠上下文隔离，使玻璃半透明组件（PlayerBar/SideBar）能穿透显示底层光晕
- **参考**：Aceternity UI Background Gradient Animation（交互渐变光晕概念）
- **实际变更**：

  | 文件 | 修改内容 |
  |------|---------|
  | `AmbientGlow.vue` | **新建** Canvas 光晕渲染组件；`position: absolute; z-index: -1`；光晕强度 0.8~1.8（5x 增强后） |
  | `App.vue` | 引入 AmbientGlow，挂载到 `#app-container` 首个子元素 |
  | `base.css` | `#app-container` 移除 `z-index: 1`；`::after` 移除 `z-index: -1` |
- **涉及文件**（共 3 个）：
  - `sys_vue/src/components/background/AmbientGlow.vue`（新建）
  - `sys_vue/src/App.vue`
  - `sys_vue/src/styles/base.css`
- **验收标准**：背景呈现流动光晕效果，鼠标移动时光晕跟随产生流体感，PlayerBar/SideBar 玻璃区域可见光晕，CPU 占用 < 5%

- **方案**：为结果容器添加 `transition-group`，行添加 `fade-in-up` 逐个入场
- **涉及文件**：
  - `sys_vue/src/components/pages/DiscoverMusic.vue`
  - `sys_vue/src/components/business/BaseSongTable.vue`
- **验收标准**：翻页/搜索时歌曲行逐个淡入上浮

#### P3-3：LyricPage 歌词分层动画

- **问题**：当前歌词行无特殊动画，非当前行无视觉区分
- **方案**：当前行添加渐入缩放 + 辉光效果，非当前行降低 opacity + 微缩，参考 Mineradio 歌词层级（readability → glow → sun）
- **涉及文件**：
  - `sys_vue/src/components/pages/LyricPage.vue`
- **验收标准**：当前歌词行有突出动画，前后行自然淡出

#### P3-4：专辑封面色扩散系统

- **问题**：`useAlbumColors.ts` 已提取主色，但仅用于底部光晕
- **方案**：将主色扩散到侧边栏选中指示器、PlayerBar 进度条、高亮色等
- **涉及文件**：
  - `sys_vue/src/composables/useAlbumColors.ts`
  - `sys_vue/src/components/system/SideBar.vue`
  - `sys_vue/src/components/system/PlayerBar.vue`
- **验收标准**：播放歌曲后，UI 高亮色跟随封面主色变化

#### P3-5：MusicLibrary DOM 操作修复

- **问题**：`classList.add('highlight-update')` 和 `textContent` 直接操作 DOM
- **方案**：改为响应式 `ref` + computed 驱动的 CSS class 绑定
- **涉及文件**：
  - `sys_vue/src/components/pages/MusicLibrary.vue`
- **验收标准**：无原生 DOM 操作，全响应式驱动
